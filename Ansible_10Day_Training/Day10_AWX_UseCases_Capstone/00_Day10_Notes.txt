PART A - ANSIBLE AWX  (web UI / API on top of Ansible)
----------------------------------------------------------------
  AWX is the open-source upstream of Ansible Automation Platform
  (Tower). It gives a web UI, REST API, RBAC and scheduling on top
  of the same playbooks we have been writing.

  CORE COMPONENTS
    1. Projects        - where playbooks live (synced from GitHub)
    2. Inventories     - hosts and groups
    3. Credentials     - SSH keys / passwords (stored securely)
    4. Job Templates   - a blueprint to run a playbook
                         = project + inventory + credentials + playbook
    5. Schedules       - run jobs at set times (like cron)
    6. Execution Env   - containerised Ansible runtime images

  TYPICAL WORKFLOW
    1. Admin pushes playbooks to GitHub
    2. AWX syncs the project from GitHub
    3. User selects a Job Template
    4. AWX connects to the target hosts using stored credentials
    5. AWX runs the playbook inside an execution environment
    6. AWX shows results and logs in the UI

  INSTALL (high level)
    AWX is installed with the AWX Operator on Kubernetes
    (k8s / OpenShift / minikube).
